# 📋 פנקס המאמן — אפיון מוצר ומערכת עיצוב (SPEC)

> מסמך עבודה לפיתוח מתמשך. ההווה מתועד, העתיד מוגדר, והעיצוב מאוחד תחת מערכת אחת. כל פיצ'ר חדש צריך לציית למערכת העיצוב כאן.

---

## 1. סקירה כללית
**מה זה:** אפליקציית ווב חברתית-מקצועית למאמני כדורסל בעברית (RTL). פלטפורמה שבה מאמן בונה זהות מקצועית, מנהל את הידע שלו (תרגילים ותוכניות), מארגן את העבודה השוטפת (לו"ז), ומתחבר לקהילת מאמנים.

**הבעיה שהיא פותרת:** הידע של מאמני כדורסל מפוזר — במחברות, בראש, בקבוצות וואטסאפ. אין מקום אחד שמרכז תרגילים, בונה מהם תוכניות אימון, מנהל לו"ז קבוצות, ומחבר בין מאמנים. פנקס המאמן הוא ה"פנקס הדיגיטלי" הזה.

**הצעת הערך המרכזית:** ספריית תרגילים שיתופית של הקהילה + בונה תוכניות חכם + ניהול שוטף, הכול במקום אחד ובעברית.

## 2. עקרונות מנחים
**עקרונות מוצר:**
- **מהיר על המגרש.** מאמן משתמש באפליקציה לפעמים תוך כדי אימון — כל זרימה מרכזית חייבת להיות קצרה ולעבוד טוב בטלפון.
- **קהילתי כברירת מחדל.** תרגילים, דירוגים ותגובות הם משאב משותף; הפרטי (מועדפים, תוכניות, לו"ז) נשאר פרטי.
- **עברית ו-RTL הם אזרחים סוג א'.** לא תרגום של ממשק לועזי — חוויה שנבנתה לעברית מהיסוד.

**עקרונות פיתוח (סגנון העבודה מול קלוד קוד):**
- הסבר כל צעד **בעברית**, פשוט וברור — המשתמש מתחיל בקוד.
- **צעד אחד בכל פעם** — לתת צעד, לחכות לאישור, להמשיך. לא להציף.
- **לבצע את העריכות בעצמך** בקבצים.
- כל SQL חדש = **קובץ נפרד** עם הסבר, מסתיים ב-`notify pgrst, 'reload schema';`.
- אחרי כל שינוי קוד — `npm run build` לוודא שאין שגיאות.
- **כל UI חדש מציית למערכת העיצוב בסעיף 8.** לא להמציא צבעים, מרווחים או רכיבים חדשים — להשתמש בטוקנים הקיימים.

## 4. מצב נוכחי — מה כבר בנוי ועובד

### 4.1 פרופילים וקהילה
- הרשמה / התחברות + איפוס סיסמה.
- פרופיל מאמן: שם, מועדון, שכבות גיל.
- מאתר מאמנים: חיפוש וסינון.
- פרופיל מאמן לחיץ: צפייה בתרגילים של מאמן אחר.
- "האזור שלי": סטטיסטיקות אישיות.

### 4.2 ספריית תרגילים
- הוספת תרגיל עם שדות עשירים: קטגוריה, שכבות גיל, משך, ציוד, רמת קושי, מטרה, חזרות, דגשים, סרטון.

- דירוג בכוכבים, שמירה למועדפים, תגובות.

### 4.3 תוכניות אימון
- בונה ידני: סדר, משך והערה לכל תרגיל; פתיחת פרטים מלאים.

- מצב טיימר להרצה על המגרש.

### 4.4 לו"ז
- לו"ז שבועי לקבוצות (יום / שעה / קבוצה) + צירוף תוכנית אימון לכל שורה.

### 4.5 תקשורת
- הודעות פרטיות 1-על-1.
- צ'אט קבוצתי לכל המאמנים.
- לוח משחקי אימון: פרסום בקשה + שליחת הודעה.

## 8. 🎨 מערכת העיצוב (Design System)
> זו השכבה הכי חשובה לשמירה על "נקי ומסודר". המטרה: כל מסך באפליקציה נראה כאילו אדם אחד עיצב אותו. כל ערך כאן צריך להיות מוגדר פעם אחת כ-CSS variable ולהיות בשימוש בכל מקום — בלי ערכים "קשיחים" מפוזרים בקוד.

### 8.1 פילוסופיית עיצוב
ספורטיבי, נקי, מקצועי. נשען על שני צבעי המותג — **כתום אנרגטי** (פעולה, הדגשה) ו**נייבי רציני** (מבנה, עומק). הרבה מרחב נשימה, פינות מעוגלות בעקביות, היררכיה ברורה. לא עמוס.

### 8.2 צבעים (Color Tokens)
**מותג — כתום (פעולה ראשית):**
```
--orange-50:  #FDF2EC
--orange-100: #FADFCE
--orange-300: #F0A878
--orange-500: #E8763A   ← צבע המותג
--orange-600: #D15F23   ← hover
--orange-700: #A8491A   ← active/pressed
```
**מותג — נייבי (מבנה ועומק):**
```
--navy-500: #41598C
--navy-700: #2F4068
--navy-800: #243352
--navy-900: #1B2A4A   ← צבע המותג
```
**ניטרליים (אפורים):**
```
--gray-50:  #F7F8FA
--gray-100: #EEF0F4
--gray-200: #DCE0E8
--gray-300: #C2C8D4
--gray-500: #8A93A6
--gray-700: #4B5567
--gray-900: #1A1F2B
```
**סמנטיים (משוב למשתמש):**
```
--success: #2E9E5B   (הצלחה, שמירה)
--warning: #E0A92E   (אזהרה)
--error:   #D64545   (שגיאה, מחיקה)
--info:    #3A82E8   (מידע)
```
**טוקנים לפי תפקיד — מצב בהיר:**
```
--bg:           #F7F8FA   (רקע אפליקציה)
--surface:      #FFFFFF   (כרטיסים, מודלים)
--surface-alt:  #EEF0F4   (רקע משני)
--text:         #1A1F2B   (טקסט ראשי)
--text-muted:   #4B5567   (טקסט משני)
--border:       #DCE0E8
--accent:       var(--orange-500)
```
**טוקנים לפי תפקיד — מצב כהה:**
```
--bg:           #131722
--surface:      #1B2230   (≈ נייבי כהה)
--surface-alt:  #232C3D
--text:         #EEF0F4
--text-muted:   #9AA3B5
--border:       #2E3850
--accent:       var(--orange-500)   ← הכתום נשאר זהה לשמירת זהות המותג
```
> **כלל:** רכיבים אף פעם לא מתייחסים ל-`#FFFFFF` או `#1B2A4A` ישירות — רק ל-`var(--surface)`, `var(--accent)` וכו'. כך מצב כהה "פשוט עובד".

### 8.3 טיפוגרפיה
**גופן:** Rubik (תומך עברית מצוין, מודרני וספורטיבי). חלופות: Heebo / Assistant.
```
--font: 'Rubik', system-ui, sans-serif;
```
**סולם גדלים (Type Scale):**
```
--text-xs:   12px / 1.4   (תוויות, מטא)
--text-sm:   14px / 1.5   (טקסט משני)
--text-base: 16px / 1.6   (גוף ברירת מחדל)
--text-lg:   18px / 1.5   (כותרות כרטיס)
--text-xl:   22px / 1.4   (כותרות מסך)
--text-2xl:  28px / 1.3   (כותרת ראשית)
```
**משקלים:** 400 (רגיל) · 500 (בינוני, ברירת מחדל לכפתורים) · 700 (כותרות). לא יותר משלושה משקלים.

### 8.4 מרווחים ורשת (Spacing)
בסיס של **4px**. כל margin/padding/gap הוא כפולה:
```
--space-1: 4px    --space-4: 16px   --space-8: 32px
--space-2: 8px    --space-5: 20px   --space-10: 40px
--space-3: 12px   --space-6: 24px   --space-12: 48px
```
- מרווח פנימי סטנדרטי בכרטיס: `--space-5`.
- מרווח בין רכיבים ברשימה: `--space-3`.
- רוחב תוכן מקסימלי במסכי תוכן: ~1100px, ממורכז.

### 8.5 פינות, צללים וגבולות
```
--radius-sm:  8px    (כפתורים, אינפוטים)
--radius-md:  12px   (כרטיסים)
--radius-lg:  20px   (מודלים)
--radius-full: 999px (אווטאר, צ'יפים)
--shadow-sm: 0 1px 2px rgba(27,42,74,.06)
--shadow-md: 0 4px 12px rgba(27,42,74,.10)
--shadow-lg: 0 12px 32px rgba(27,42,74,.16)
```
במצב כהה הצללים עדינים יותר (אטימות נמוכה) או מוחלפים בגבול דק.

### 8.6 רכיבי UI (קטלוג)
- **כפתורים:** ראשי (רקע `--accent`, טקסט לבן, `--radius-sm`, hover `--orange-600`) · משני (שקוף, גבול `--border`) · רפאים (ללא רקע/גבול) · סכנה (`--error`, רק להרסני). גובה אחיד 44px.
- **כרטיס:** `--surface`, `--radius-md`, `--shadow-sm`, padding `--space-5`.
- **שדה קלט:** גובה 44px, `--radius-sm`, גבול `--border`, פוקוס → `--accent` + הילה.
- **צ'יפ/תגית:** `--radius-full`, רקע `--surface-alt`, טקסט קטן.
- **דירוג כוכבים:** מלא `--accent`, ריק `--gray-300`.
- **מודל:** `--surface`, `--radius-lg`, `--shadow-lg`, רקע-על כהה. נסגר ב-Esc וקליק בחוץ.
- **טוסט:** משוב קצר (נשמר/נמחק/שגיאה) לפי צבע סמנטי.
- **טאבים:** הטאב הפעיל מסומן בקו תחתון `--accent`.
- **מצב ריק:** אייקון עדין + משפט מנחה + כפתור פעולה.
- **שלד טעינה:** לרשימות במקום ספינר.

### 8.7 כללי RTL
- `dir="rtl"`, כל הלייאאוט מימין לשמאל.
- תכונות לוגיות: `margin-inline-start` ולא `margin-left`, `padding-inline` וכו'.
- אייקונים כיווניים (הבא/הקודם) מתהפכים אופקית.
- מספרים, שעות וסרטונים — יישור נכון בתוך זרימת RTL.

### 8.8 נגישות
- ניגודיות AA (4.5:1). לטקסט קטן על רקע בהיר להעדיף `--navy-900` או `--orange-700`.
- כל שדה עם `<label>`. כל אייקון-כפתור עם `aria-label`.
- מצבי פוקוס נראים. יעדי מגע ≥ 44×44px.

### 8.9 תנועה
- מעברים עדינים: 150–200ms, `ease-out`. כיבוד `prefers-reduced-motion`.

### 8.10 אייקונוגרפיה
- ספריית אייקונים אחת ועקבית (למשל `lucide-react`) — קו ועובי אחידים.

## 9. זרימות משתמש מרכזיות
1. **מציאת תרגיל ושמירתו:** תרגילים → סינון → פתיחת כרטיס → ⭐ דירוג / 🔖 שמירה.
2. **בניית אימון מהיר:** תוכניות → בנאי חכם → גיל+נושאים+זמן → תוצאה → התאמות → שמירה.
3. **הרצת אימון:** תוכנית → מצב טיימר → מעבר תרגיל-תרגיל.
4. **ניהול שבוע:** לו"ז → הוספת שורה → צירוף תוכנית.
5. **חיבור לקהילה:** מאתר מאמנים → פרופיל → תרגילים שלו / שליחת הודעה.

## 12. המלצות להוספה
1. **קובץ טוקנים מרוכז (CSS variables)** — הצעד הראשון לפני כל פיצ'ר עיצובי. ממיר את הצבעים והמרווחים הפזורים לטוקנים אחידים.
2. **רכיבי בסיס משותפים** — `Button`, `Card`, `Input`, `Badge`, `Modal` כרכיבים יחידים.
