# 🏀 פנקס המאמן — מדריך התקנה לשלב 1

ברוך הבא! המדריך הזה ילווה אותך צעד-אחר-צעד עד שיהיה לך אתר חי באינטרנט.
אין צורך בידע מוקדם. פשוט עקוב אחרי השלבים **בדיוק לפי הסדר**.

> 💡 **טיפ:** קרא כל שלב עד הסוף לפני שאתה מתחיל לבצע אותו.

---

## מה מותקן ומה נתקין

✅ כבר התקנת: **Node.js** ו-**VS Code** (מצוין!)
⬜ נתקין בהמשך: חשבון **Supabase** (מסד נתונים) וחשבון **Netlify** (אירוח). שניהם חינמיים.

**סדר העבודה:**
1. פתיחת הפרויקט ב-VS Code
2. התקנת רכיבי הפרויקט (פקודה אחת)
3. הקמת מסד הנתונים ב-Supabase
4. חיבור הפרויקט ל-Supabase
5. הרצת האתר על המחשב שלך
6. העלאת האתר לאינטרנט

---

## שלב 1: פתיחת הפרויקט ב-VS Code

1. **חלץ את קובץ ה-ZIP** שהורדת (לחיצה ימנית → "חלץ הכול" / Extract All). תקבל תיקייה בשם `pinkas-hamaman`.
2. שמור אותה במקום נוח, למשל בשולחן העבודה.
3. פתח את **VS Code**.
4. בתפריט העליון: **File → Open Folder** (קובץ → פתח תיקייה).
5. בחר את התיקייה `pinkas-hamaman` ולחץ "בחר תיקייה".

אתה אמור לראות בצד שמאל את כל קבצי הפרויקט. 🎉

---

## שלב 2: התקנת רכיבי הפרויקט

עכשיו נפתח את ה"טרמינל" — חלון שבו מקלידים פקודות.

1. ב-VS Code, בתפריט העליון: **Terminal → New Terminal** (טרמינל → טרמינל חדש).
2. בתחתית המסך ייפתח חלון. שם נקליד פקודות.
3. **הקלד בדיוק את הפקודה הבאה** ואז לחץ Enter:

```
npm install
```

4. **מה אמור לקרות:** יופיעו שורות טקסט, וזה ייקח כדקה-שתיים. בסוף תראה שורה שמסתיימת בערך כך: `added XXX packages`. זהו — הצליח.

> ⚠️ אם הופיעה שגיאה אדומה, צלם מסך ושלח לי.

---

## שלב 3: הקמת מסד הנתונים ב-Supabase

מסד הנתונים הוא ה"מחסן" שבו נשמרים המשתמשים והמידע.

### 3א. פתיחת חשבון

1. היכנס לאתר: **https://supabase.com**
2. לחץ **Start your project** והירשם (הכי קל דרך חשבון GitHub, או עם מייל).
3. אחרי ההרשמה, לחץ **New Project** (פרויקט חדש).
4. מלא:
   - **Name:** `pinkas-hamaman` (או כל שם)
   - **Database Password:** בחר סיסמה חזקה ו**שמור אותה** במקום בטוח (לא נצטרך אותה כרגע, אבל חשוב לא לאבד).
   - **Region:** בחר `Central EU (Frankfurt)` — הכי קרוב לישראל.
5. לחץ **Create new project**.
6. **המתן כ-2 דקות** עד שהפרויקט נבנה (יופיע מסך טעינה).

### 3ב. הרצת קובץ ה-SQL

1. בתפריט הצדדי השמאלי של Supabase, לחץ על האייקון **SQL Editor** (נראה כמו `</>`).
2. לחץ **New query** (שאילתה חדשה).
3. פתח את הקובץ `supabase_setup.sql` מתיקיית הפרויקט (אפשר לפתוח אותו ב-VS Code), **העתק את כל התוכן** שלו.
4. **הדבק** אותו לתוך החלון של Supabase.
5. לחץ על הכפתור **Run** (או Ctrl+Enter).
6. **מה אמור לקרות:** למטה תופיע הודעה ירוקה `Success. No rows returned`. מצוין — מסד הנתונים מוכן! ✓

## שלב 4: חיבור הפרויקט ל-Supabase

עכשיו נחבר בין הקוד שלך למסד הנתונים, באמצעות שני "מפתחות".

### 4א. השגת המפתחות מ-Supabase

1. ב-Supabase, בתפריט הצדדי, לחץ על אייקון **גלגל השיניים** (Settings) למטה.
2. לחץ על **API** (או **Data API**).
3. תראה שני ערכים שנעתיק:
   - **Project URL** — כתובת שמתחילה ב-`https://...supabase.co`
   - **anon public** key — מחרוזת ארוכה של אותיות ומספרים (תחת "Project API keys").

> ⚠️ חשוב: השתמש במפתח **anon / public** בלבד. **אל** תשתמש במפתח `service_role` — הוא סודי ולא מיועד לאתר.

### 4ב. יצירת קובץ ההגדרות

1. חזור ל-VS Code.
2. בצד שמאל, בתוך תיקיית הפרויקט, תמצא קובץ בשם `.env.local.example`.
3. **צור קובץ חדש** בשם **`.env.local`** (בדיוק כך, כולל הנקודה בהתחלה):
   - לחץ ימני על שם הפרויקט → **New File** → הקלד `.env.local` → Enter.
4. העתק לתוכו את שתי השורות הבאות, והחלף את הערכים במה שהעתקת מ-Supabase:

```
VITE_SUPABASE_URL=הדבק_כאן_את_ה-Project_URL
VITE_SUPABASE_ANON_KEY=הדבק_כאן_את_מפתח_ה-anon
```

**דוגמה** (הערכים שלך יהיו שונים):
```
VITE_SUPABASE_URL=https://abcdxyz123.supabase.co
VITE_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6...
```

5. **שמור** את הקובץ (Ctrl+S).

> 💡 שים לב: אין רווחים מסביב לסימן `=`, ואין מרכאות.

## שלב 5: הרצת האתר על המחשב שלך

רגע האמת הראשון!

1. בטרמינל (אם נסגר — פתח שוב: Terminal → New Terminal), הקלד:

```
npm run dev
```

2. **מה אמור לקרות:** יופיע טקסט עם כתובת, בערך כך:

```
  ➜  Local:   http://localhost:5173/
```

3. החזק **Ctrl** ולחץ על הכתובת `http://localhost:5173/` (או העתק אותה לדפדפן).
4. ייפתח האתר! אתה אמור לראות את מסך ההתחברות של **פנקס המאמן**. 🎉

### בדיקה שהכול עובד:

1. לחץ על הטאב **הרשמה**.
2. הזן מייל אמיתי שלך וסיסמה (לפחות 6 תווים), ולחץ **יצירת חשבון**.
3. אמורה להופיע הודעה ירוקה: "נרשמת בהצלחה! בדוק את תיבת המייל...".
4. **בדוק את המייל שלך** — אמור להגיע מייל אישור מ-Supabase. לחץ על הקישור שבו.
5. חזור לאתר, עבור לטאב **התחברות**, הזן את אותם פרטים ולחץ **כניסה**.
6. אם נכנסת ורואה "ברוך הבא לפנקס המאמן!" — **שלב 1 עובד באופן מלא!** 🏀✓

> ⚠️ אם לא הגיע מייל: לפעמים זה בתיקיית הספאם. אפשר גם להיכנס ב-Supabase ל-**Authentication → Users** ולראות שהמשתמש שלך נוצר.

> 🛑 **לעצור את האתר:** בטרמינל, לחץ `Ctrl + C`. כדי להפעיל שוב — `npm run dev`.

## שלב 6: העלאת האתר לאינטרנט (Netlify)

עד עכשיו האתר רץ רק על המחשב שלך. עכשיו ניתן לו כתובת אמיתית שכל אחד יוכל לגשת אליה.

הדרך הכי פשוטה היא **גרירה ידנית** של תיקיית הבנייה:

### 6א. בניית גרסה לאינטרנט

1. בטרמינל (עצור קודם את האתר עם `Ctrl + C`), הקלד:

```
npm run build
```

2. **מה אמור לקרות:** נוצרת תיקייה חדשה בשם `dist` בתוך הפרויקט. זו הגרסה המוכנה לאינטרנט.

### 6ב. העלאה ל-Netlify

1. היכנס לאתר: **https://www.netlify.com** והירשם (שוב, הכי קל עם GitHub או מייל).
2. אחרי ההתחברות, חפש את האזור **Sites**.
3. תראה אזור עם הכיתוב **"Drag and drop your site output folder here"** (גרור ושחרר כאן).
4. פתח את סייר הקבצים של Windows, היכנס לתיקיית הפרויקט `pinkas-hamaman`, ומצא את התיקייה **`dist`**.
5. **גרור את תיקיית `dist`** ושחרר אותה על האזור ב-Netlify.
6. **המתן כ-30 שניות.** Netlify יעלה את האתר וייתן לך כתובת אמיתית, משהו כמו: `https://random-name-123.netlify.app`.
7. לחץ על הכתובת — **האתר שלך חי באינטרנט!** 🌍🎉

> 💡 אפשר לשנות את השם המוזר: ב-Netlify → **Site configuration → Change site name**.

> ⚠️ **חשוב:** אחרי שהעלית, נצטרך עוד צעד קטן ב-Supabase כדי שההרשמה תעבוד גם באתר החי (הוספת הכתובת ל"רשימה המאושרת"). תגיד לי כשהגעת לכאן ואני אסביר — זה 30 שניות.
